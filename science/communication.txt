2. Rest API URIs

-   get /api/info
    info about this rest api server

-   /api/auth
    authentication & authorization stuff
    -   post /login
    -   post /logout
    -   post /refresh

-   /api/person
    user profile management
    -   post /create
    -   post /update
    -   get /view?id={id}

-   /api/project
    project management
    -   post /create
    -   post /update
    -   post /transfer
    -   get /view?id={id}
    -   get /list?page={page}
    -   delete /delete?id={id}
    -   post /clone

-   /api/projectinvolvement
    project involvement management
    -   post /create
    -   post /update
    -   get /view?personId={personId}&projectId={projectId}
    -   get /list?page={page}&projectId={projectId}&personId={personId}
    -   delete /delete?personId={personId}&projectId={projectId}

-   /api/plugin
    plugin management
    -   post /create
    -   post /update
    -   get /view?id={id}
    -   get /list?page={page}&query={query}&authorId={authorId}

3. WebSocket

-   /ws
    connect to one websocket endpoint

-   auth (personId)
    auth as person with given id

-   subscribe (projectId)
    subscribe to given project

-   patch (projectId, data)
    patch project data

-   request (projectId, topic, page)
    request for commits or event log or plugins
